# Lint Skill

`practice-scenario/src/`의 Python 코드 품질을 점검합니다. 코드 수정은 하지 않습니다.

## 수행 절차

1. `practice-scenario/src/app.py`와 `practice-scenario/src/store.py`를 읽습니다
2. 아래 3가지 항목을 점검합니다

## 점검 항목

### 1. ID 검증 중복
- `todo_id < 1` 검증이 여러 엔드포인트에 반복되는지 확인
- 개선: FastAPI `Path(ge=1)` 또는 공통 함수로 추출

### 2. 제목 검증 중복
- 제목 빈값/길이 검증이 create와 update에 반복되는지 확인
- 개선: Pydantic `field_validator`로 모델에 통합

### 3. 불필요한 수동 타입 체크
- `isinstance(req.title, str)` 등 Pydantic이 이미 처리하는 타입을 수동 검증하는지 확인
- `isinstance(req.completed, bool)` 등 동일한 패턴
- 개선: Pydantic 모델이 타입 검증을 자동 처리하므로 제거


## 출력 형식

```
## 코드 품질 점검 결과

### 🔧 중복 코드 (N건)
- [파일:라인] 설명
  → 개선 방안

### ⚠️ 불필요한 코드 (N건)
- [파일:라인] 설명
  → 개선 방안

### 점수: N/100
- ID 검증 중복: -5점 × N건 = -N점
- 제목 검증 중복: -5점 × N건 = -N점
- 불필요한 타입 체크: -3점 × N건 = -N점
```

## 점수 기준
- 100점에서 시작하여 발견 건수 × 건당 감점으로 계산
- ID 검증 중복: 건당 -5점 (예: 3건 → -15점)
- 제목 검증 중복: 건당 -5점 (예: 2건 → -10점)
- 불필요한 수동 타입 체크: 건당 -3점 (예: 3건 → -9점)
